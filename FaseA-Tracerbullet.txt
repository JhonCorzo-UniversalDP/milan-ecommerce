     Tracer Bullet: Catalog → Product → Add to Cart → Cart

     Context

     The repo has placeholder pages for /, /product/[slug], and /cart (all
     TODOs), a Prisma schema with Product, User, CartItem, and a seed of 20 bike
      products. Goal: implement the thinnest possible end-to-end vertical slice
     ("Phase A" per AGENTS.md) that walks a user from catalog → product detail →
      add to cart → cart view, and back this slice with a single tracer-bullet
     test in Vitest.

     This is intentionally minimal — no styling polish, no quantity controls, no
      auth, no error UI. The point is to prove the wiring across UI, server
     actions, and Prisma works end-to-end so subsequent vertical features
     (search, recommendations, MCP-backed catalog in Phase B) can plug in.

     Repository setup (before any code changes)

     The working directory is not a git repo yet. Before implementing anything:

     1. git init in /home/jhon_corzo/ClaudeCourse/CompetenciaFinal.
     2. Confirm .gitignore already excludes node_modules, .next, prisma/dev.db,
     .env (it does — verify and amend if needed).
     3. gh repo create milan-ecommerce --public --source=. --remote=origin (no
     README/license to avoid initial-merge churn).
     4. Commit 1 (scaffold): git add everything currently in the repo, commit as
      chore: initial scaffold, push to origin main.
     5. Then implement the tracer slice — that produces Commit 2 (tracer:)
     below, with a clean isolated diff.

     Final commit

     After implementation + tests + lint pass, create one commit on main:
     - Subject: tracer: end-to-end catalog → product → add to cart → cart slice
     - Body: bullet list of what's wired (lib modules, pages, server action,
     tracer test).
     - Push to origin main.

     Scope

     In:
     - Catalog list of all seeded products at / linking to /product/[slug]
     - Product detail at /product/[slug] showing name + "Agregar al carrito"
     form
     - Server action that creates an anonymous cart user (cookie-based), inserts
      a CartItem, and redirects to /cart
     - Cart page listing items by product name
     - One tracer-bullet Vitest integration test that exercises the full flow

     Out (deferred to Phase B / later commits):
     - Auth/login, quantity controls, remove-from-cart, prices/styling, Odoo MCP
      integration, search, recommendations.

     Design — thin UI, fat /lib/

     Per AGENTS.md: Prisma stays in /lib/db/*; components consume domain types
     only.

     Domain types

     Define once, reused across layers (place in lib/types.ts):
     export type Product = { id: string; slug: string; name: string };
     export type CartLine = { productId: string; name: string; quantity: number
     };
     (Keep narrow — only fields the tracer slice needs. Price/description added
     later.)

     New /lib/ modules

     lib/db/catalog.ts — sole module that reads Product.
     - listProducts(): Promise<Product[]> — prisma.product.findMany({ orderBy: {
      name: "asc" } }), maps to Product.
     - getProductBySlug(slug: string): Promise<Product | null> —
     prisma.product.findUnique({ where: { slug } }), maps to Product.

     lib/db/cart.ts — sole module that reads/writes CartItem and the anonymous
     User.
     - getOrCreateAnonymousUserId(): Promise<string> — reads cartUserId cookie
     via next/headers; if absent, creates a User with a synthetic email (e.g.
     anon-<cuid>@local), sets the cookie (httpOnly, 30d), returns the new id.
     - addProductToCart(slug: string): Promise<void> — resolves user via
     getOrCreateAnonymousUserId, looks up product id by slug, upserts a CartItem
      on the @@unique([userId, productId]) constraint (increment quantity on
     conflict).
     - listCartLines(): Promise<CartLine[]> — resolves user (returns [] if no
     cookie), joins CartItem with Product, maps to CartLine[].

     Reuses existing lib/prisma.ts singleton.

     Pages and server action

     app/page.tsx (server component)
     - const products = await listProducts();
     - Renders <ul> of <a href={/product/${p.slug}}>{p.name}</a>.

     app/product/[slug]/page.tsx (server component)
     - const product = await getProductBySlug(slug); → notFound() if null.
     - Renders <h1>{product.name}</h1> and a <form action={addToCartAction}>
     with a hidden slug input and a submit button labeled "Agregar al carrito".

     app/product/[slug]/actions.ts ("use server")
     - export async function addToCartAction(formData: FormData):
       - reads slug from formData,
       - calls addProductToCart(slug),
       - redirect("/cart") from next/navigation.

     app/cart/page.tsx (server component)
     - const lines = await listCartLines();
     - Empty state: <p>Carrito vacío</p>.
     - Non-empty: <ul> of <li>{name} × {quantity}</li>.

     Tracer-bullet test

     tests/tracer.test.ts — single integration test exercising the slice without
      spinning up Next.

     Strategy: import the /lib/db/* functions directly (the routes are thin
     wrappers around them, per AGENTS.md). Mock next/headers cookies() to expose
      a writable in-memory cookie jar so getOrCreateAnonymousUserId works. Use
     the real seeded SQLite prisma/dev.db.

     Test body (single it so it reads like the user story):
     1. const products = await listProducts(); → assert it includes a known seed
      slug (e.g. milan-urbana-01).
     2. const product = await getProductBySlug("milan-urbana-01"); → assert
     non-null, name matches.
     3. await addProductToCart("milan-urbana-01"); → no throw.
     4. const lines = await listCartLines(); → assert one line with name: "Milán
      Urbana 01", quantity: 1.

     Teardown: delete the anonymous User (cascades to CartItems) so re-runs are
     idempotent.

     Note: the route components themselves are not exercised by Vitest —
     verifying they render is trivial once the lib layer is proven. The actual
     /, /product/[slug], /cart HTTP wiring is verified by running pnpm dev and
     clicking through, which is the human "tracer" half of the bullet.

     Files to create / modify

     Create:
     - lib/types.ts
