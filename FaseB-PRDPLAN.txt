### Catalog view                                                                
In the "/" endpoint create a list for showing 50 products just display the name 
 and price of each product. When the user clicks on the row, the page must take 
 the user to the product view.                                                  
                                                                                
### Product view                                                                
In this view the user will only see the product name, description, price and    
barcode. Add a button for adding the product to the cart                        
                                                                                
#### Suggested products                                                         
This feature will be placed inside the product view, I want to keep it basic so 
 just return the next 5 items that have a similar name to the product the user  
opened

database so by now just use the current test data located in

For this particular feature create a basic form where only a user email is      
asked, the only validation needed is that it must be a valid email. Store in

 localstorage, just use the easiest and fastest

● PRD written to docs/prds/001-catalog-mvp.md.

  Key decisions encoded:
  - Catalog adapter (/lib/catalog/*) is the only place that knows the data
  source; default = JSON adapter reading data/product_template.json, swappable
  to Odoo MCP via CATALOG_SOURCE env var.
  - URLs use product id (not slug) since the source JSON has no slug.
  - Cart + session in localStorage (per "easiest/fastest" hint); Prisma CartItem
   stays dormant.
  - Similarity = pure Jaccard over normalized tokens — testable, replaceable.
  - Tests scoped to similarity, jsonAdapter, email validator, cart.addItem.
  - Prices stored as priceCents, rendered as COP.
  - Search is a server-side GET ?q= form on / using the same adapter
  (LIKE-style).

  Out of scope explicitly: DB-persisted cart, real auth, checkout, images,
  pagination beyond 50, Odoo wiring itself.

  polish:

  1. Catalog adapter + / listing — foundational module + 50-item list.
